# 嘟嘟动画效果 — 完整清单 & 生成提示词

## 一、画风建议

当前用的是写实风生成图。**批量生成 sprite 动画，写实风几乎不可能保持一致性**——每张生成的猫都会长得不一样，帧与帧之间会闪烁/跳变。

| | 写实风（当前） | **卡通矢量/扁平风（推荐）** |
|---|---|---|
| 跨帧一致性 | 差，每张脸不同 | 好，特征可锁定 |
| 批量可行性 | 低，需要大量重试 | 中高 |
| 桌宠适用性 | 一般 | 更好——可爱、轻量、讨喜 |
| GPT 生成难度 | 高 | 中 |

**推荐风格**：干净的卡通渲染风（类似《猫咪后院》/《旅行青蛙》的扁平插画风），蓝色英短特征保留（圆脸、灰蓝毛色、铜色眼睛、短耳），但线条简化、色块分明。或者退一步，像素风（`pixel art`）——对一致性要求最低，GPT 也能生成。

## 三、给 GPT 的图片生成提示词

以下是一个**主提示词模板**，可以直接发给 GPT（DALL·E / GPT-4o 图像生成）：

```
你是一个游戏 sprite 动画师。请为一只叫"嘟嘟"的猫咪桌宠生成 sprite sheet 帧序列。

## 角色设定

一只英国短毛蓝猫，卡通扁平插画风（类似猫咪后院/旅行青蛙的柔和矢量风格）：
- 圆脸，脸颊饱满
- 灰蓝色短毛，毛色均匀
- 铜色/琥珀色圆眼睛
- 小短耳，耳尖微圆
- 短粗身材，微胖可爱
- 尾巴粗短，尾尖微圆


## 技术要求

- 每帧 512×512，背景必须完全透明（PNG）
- 所有帧必须是同一只猫，保持角色设计完全一致
- 猫在画面中央，不要裁剪
- 按时间顺序从左到右排列成水平条带（sprite sheet）
- 或者每帧单独一张图，按序号命名

## 动画列表

请按照以下规格，为每个动画生成 sprite sheet：

### 1. idle（待机呼吸）- 循环
帧数：10 帧，帧率 3fps
画面：猫侧坐，面向右偏 45°。身体随呼吸轻柔起伏（2-3 像素的膨胀收缩），尾巴尖偶尔轻微晃动。表情放松，偶尔慢眨眼。

### 2. sleep（睡觉）- 循环
帧数：8 帧，帧率 3fps
画面：猫蜷成一个完美的圆球，像一颗蓝灰色大福团子。轻微呼吸起伏。安静、治愈。

### 3. walk（走路）- 循环
帧数：6 帧，帧率 8fps
画面：侧视角，短腿交替前移，身体有弹跳般的上下颠簸感。胖乎乎的身体一颠一颠的。

### 4. excited（兴奋跳转圈）- 单次
帧数：14 帧，帧率 10fps
画面：下蹲蓄力→高高跳起→空中旋转一圈（360°）→四肢着地站稳。表情开心，眼睛发亮。

### 5. talking（对话张嘴）- 循环
帧数：4 帧，帧率 4fps
画面：保持坐姿，嘴巴一张一合（像在喵喵叫）。闭→微张→全张→微张→闭。可加偶尔眨眼帧。

### 6. lick_mouth（舔嘴）- 单次
帧数：10 帧，帧率 8fps
画面：粉舌头伸出→舔上嘴唇→收回→抿嘴→满足表情。像刚喝完水或吃完好吃的。

### 7. wiggle_butt（扭屁股）- 循环
帧数：10 帧，帧率 5fps
画面：站立姿势，前腿伸直撑地，屁股撅高。屁股左右摇摆（幅度大，夸张可爱）。猫脸朝前或略侧，表情调皮。

### 8. petted（被摸头）- 单次
帧数：8 帧，帧率 8fps
画面：猫被摸头。眼睛眯成弯线（享受表情），头微微扬起，往一侧蹭。身体微微下沉，像被摸得很舒服。

### 9. bite（啃咬）- 单次
帧数：8 帧，帧率 10fps
画面：猫身体前倾→张嘴露出小尖牙→往前一扑咬下去→退回。咬的方向朝前（朝向屏幕/主人）。表情凶萌，但明显是嬉闹不是真攻击。

### 10. sneak_peek（猫猫祟祟）- 循环
帧数：6 帧，帧率 3fps
画面：猫从画面边缘（左侧或右侧）探出半个脑袋。先露出耳朵→再露出一只眼睛→缩回去一点→又露出来。鬼鬼祟祟。被发现了会僵住。

### 11. fluff_up（炸毛）- 单次
帧数：8 帧，帧率 8fps
画面：猫身体膨胀变大（毛炸开），背部拱起像万圣节猫，尾巴炸成鸡毛掸子。表情凶（竖瞳、呲牙）。持续几帧后慢慢恢复。

### 12. clingy_rub（贴贴蹭蹭）- 单次
帧数：10 帧，帧率 8fps
画面：猫身体软塌塌地往一侧蹭过去，眼睛眯成弯线，表情甜蜜黏糊。像在蹭屏幕/蹭人。身体拉长，然后缩回。超级软萌。

### 13. slap（猫爪拍击）- 单次
帧数：6 帧，帧率 10fps
画面：猫举起一只前爪→快速拍下→收回。动作干脆。表情不耐烦，像在打人（但萌）。

### 14. stare（凝视）- 循环
帧数：6 帧，帧率 4fps
画面：猫蹲坐，身体微微前倾，眼睛瞪大（瞳孔放大），尾巴在身后缓慢左右摆。只有尾巴动，其他部位静止。压迫感中带着可爱。

### 15. tail_flick（甩尾巴）- 循环
帧数：6 帧，帧率 6fps
画面：猫保持坐姿但一脸不耐烦，尾巴尖快速抖动/甩动。身体其他部位静止。

## 生成顺序

请先只生成 #1 idle 动画，确认风格和一致性后，再批量生成其余的。
```

## 四、验证管线

建议先跑通一个动画的全流程再批量：

```
1. 用上面提示词让 GPT 生成 idle 的 sprite sheet
2. ffmpeg -i idle.png -vf "crop=512:512:0:0" frame_%03d.png  拆帧
3. 在 Godot 里创建 SpriteFrames，导入帧，播放看效果
4. 确认帧间一致性、大小、透明背景都没问题
5. 再一次性生成其余 14 个
```
